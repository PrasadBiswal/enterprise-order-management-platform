# Enterprise Order Management Platform

## Overview
A production-grade Order Management Platform built using .NET 8, Clean Architecture, and Azure-native CI/CD pipelines.

This project simulates a real-world enterprise backend system used for managing orders, customers, and inventory in a scalable, cloud-native environment.

## Key Objectives
- Apply Clean Architecture principles in .NET
- Implement CI/CD using GitHub Actions
- Deploy to Microsoft Azure App Service
- Follow enterprise-grade coding and DevOps practices
- Prepare for real-world backend and DevOps interview scenarios

## Technology Stack
- .NET 8 (ASP.NET Core)
- GitHub Actions (CI/CD)
- Azure App Service
- Azure SQL (later phases)
- Azure Application Insights

## Business Context

The Enterprise Order Management Platform is responsible for managing the lifecycle of customer orders in a distributed enterprise environment.

### Core Responsibilities
- Accept and validate customer orders
- Maintain order state throughout its lifecycle
- Calculate order totals
- Ensure consistency of business rules
- Expose order data to downstream systems

### Non-Responsibilities
- Payment processing (handled by a separate system)
- Inventory reservation (external service)
- User authentication (enterprise identity provider)
